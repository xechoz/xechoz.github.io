# 怎么当面试官  
毕业之后，第一次当面试官，总结一些心得。

## 目标差
作为面试官，你想知道什么？或者是说，面试官的目标是什么？  
我希望知道面试者的这些方面

1. 技术水平  
一个是深度，一个是广度。  
2. 自学能力  
自学了什么？学到了什么水平？学了什么新技术？
3. 团队意识  
Code Review 怎么执行？碰到有争议的代码，怎么处理?  
怎么跟后端协商API? 怎么跟设计协商UI实现？

## 怎么问？
两个步骤
1. 广度  
有没有更多的实现方案？有没有类似的东西？
2. 深度  
原理，优缺点

首先是一个简单的实现，随后就这个知识相关的知识点广度展开，然后深度展开。

### 举例  
我需要实现一个动画，例如平移一个`TextView`。  
有几种实现方式？(基础，广度)  
还有其他的实现方式吗？(广度，扩散思维能力)  
每种的优劣？(深度)  
某种实现方式的原理？为什么要这样？比如，ValueAnimator 的实现原理. (深度) 
动画的实现原理(深度)

## 其他

1. 推荐5本技术书籍
2. 非技术类的书
3. 常见工具的使用
    1. Git 的常见操作，merge, rebase, reset
    2. 基本的 Linux 命令行


## 问题举例

### 多线程水平
1. 怎么写一个线程安全的单例？为什么这样是线程安全的or不安全的？（基础）
2. volatile, synchronized 的用法？(基础)
2. synchronized 原理？存在的问题？(深度)

### Android 相关多线程
1. 在 非UI线程可以更新 UI 吗？ 为什么？（广度)
3. 可以在子线程调Toast，Dialog吗？为什么? (广度)
2. Handler 的实现原理（广度, 深度）

### 内存管理
1. GC 回收的流程(基础)
2. 内存泄露原因？怎么查找？怎么解决？(基础)
1. GC 算法，优劣(广度，深度)
3. 写一个缓存类（基础). 为什么这样写？GC的时候，存在什么问题？(实践能力)
